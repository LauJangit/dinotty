# 移动端终端键盘设计方案

> 专为 Claude Code / OpenCode 等 AI 编程 CLI 工具设计

## 使用场景特点

Claude Code / OpenCode 的操作模式：
- 大量方向键导航（选择菜单项、浏览 diff）
- 频繁 Esc（退出交互模式、取消操作）
- Ctrl+C 中断、Ctrl+D 确认、Ctrl+Z 挂起
- Tab 补全命令/路径
- 偶尔需要输入普通文字（给 AI 的指令）
- 几乎不需要 Alt

## 按键说明

### 行1：快捷操作键

| 按键 | 发送 | 说明 |
|------|------|------|
| `^C` | `\x03` | 中断当前进程（最高频） |
| `^D` | `\x04` | EOF / 确认退出（Claude Code 常用） |
| `^Z` | `\x1a` | 挂起进程 |
| `ESC` | `\x1b` | 退出交互模式、取消 |
| `TAB` | `\x09` | 命令/路径补全 |
| `CTRL` | — | 粘滞 Ctrl 修饰键（见下） |
| `⌨️` | — | 切换系统键盘显示/隐藏 |

### 行2：方向键（十字布局）

```
         [PgUp]  [  ↑  ]  [PgDn]
         [  ←  ] [  ↓  ]  [  →  ]
```

| 按键 | 发送 |
|------|------|
| ↑ | `\x1b[A` |
| ↓ | `\x1b[B` |
| → | `\x1b[C` |
| ← | `\x1b[D` |
| PgUp | `\x1b[5~` |
| PgDn | `\x1b[6~` |

方向键在 Claude Code 中的主要用途：
- 上下键：浏览 AI 输出的菜单选项
- 左右键：在输入行中移动光标
- PgUp/PgDn：滚动查看历史输出

### 行3：常用符号（可横向滚动）

优先顺序（按 CLI 使用频率）：

```
| / ~ ` $ # & \ ' " = ; : > < ! ? @ _ - + ( ) [ ] { }
```

## Ctrl 粘滞逻辑

```
点 [CTRL] → 按钮高亮激活，显示提示"Ctrl +"
  → 点系统键盘任意字母（如 a）→ 发送 Ctrl+A (\x01)，恢复正常
  → 点工具栏其他字母符号   → 发送对应 Ctrl 组合，恢复正常
  → 再点 [CTRL] 或超时 3s  → 取消激活
```

CTRL 激活时常用组合（配合系统键盘）：

| 字母 | 组合 | 作用 |
|------|------|------|
| a | Ctrl+A | 跳到行首 |
| e | Ctrl+E | 跳到行尾 |
| r | Ctrl+R | 历史搜索 |
| u | Ctrl+U | 清除当前行 |
| w | Ctrl+W | 删除前一个词 |
| l | Ctrl+L | 清屏 |

## 实现要点

- 工具栏 `position: fixed; bottom: 0` + `env(safe-area-inset-bottom)` 适配 iPhone 安全区
- 监听 `visualViewport` resize 事件，键盘弹出时压缩终端高度，防止被遮挡
- 所有按键用 `touchstart`（而非 `click`），消除 300ms 延迟
- 方向键支持长按连续发送（100ms 间隔），用于快速滚动菜单
- Ctrl 激活状态用背景色高亮 + 文字变色明确反馈
- 符号行可横向滑动（`overflow-x: auto; scroll-snap-type: x`）
- 多分屏时每个 pane 聚焦时工具栏共享，按键发送给当前激活 pane
